triton.jit

GREEN highlights = NEW or CHANGED relative to upstream Triton (formula / example / Returns / params).  绿色 = 相比官方原版的新增或修改。 Un-annotated clean version is on the main branch.
triton.jit(fn: T) → JITFunction[T]
triton.jit(*, version=None, repr: Callable | None = None, launch_metadata: Callable | None = None, do_not_specialize: Iterable[int | str] | None = None, do_not_specialize_on_alignment: Iterable[int | str] | None = None, debug: bool | None = None, noinline: bool | None = None) → Callable[[T], JITFunction[T]]

Decorator for JIT-compiling a function using the Triton compiler.

Note:

When a jit’d function is called, arguments are implicitly converted to pointers if they have a .data_ptr() method and a .dtype attribute.

Note:

This function will be compiled and run on the GPU. It will only have access to:

  • python primitives,

  • builtins within the triton package,

  • arguments to this function,

  • other jit’d functions

Parameters:

  • fn (Callable) – the function to be jit-compiled

  • version – an optional version string used as part of the cache key, so that bumping it forces recompilation even when the source is unchanged.

  • repr (Optional[Callable]) – an optional callable that produces a short human-readable representation of a kernel launch, used in profiling and debugging output.

  • launch_metadata (Optional[Callable]) – an optional callable returning extra metadata to attach to each launch, for use by profiling tools.

  • do_not_specialize (Optional[Iterable[int | str]]) – an iterable of parameter names or indices whose values should not be specialized on (the compiler will not generate a specialized kernel per observed value).

  • do_not_specialize_on_alignment (Optional[Iterable[int | str]]) – an iterable of parameter names or indices whose values should not be specialized on alignment.

  • debug (Optional[bool]) – if True, enable debug-mode compilation (e.g. device-side assertions); defaults to the value of the TRITON_DEBUG environment knob when None.

  • noinline (Optional[bool]) – if True, prevent this function from being inlined into its callers.

Returns:

the decorated function wrapped as a JITFunction (or the decorator itself when called with keyword arguments only).

Return type:

JITFunction[T]

Example:

@triton.jit
def add_kernel(x_ptr, y_ptr, out_ptr, n, BLOCK: tl.constexpr):
    pid = tl.program_id(axis=0)
    offs = pid * BLOCK + tl.arange(0, BLOCK)
    mask = offs < n
    x = tl.load(x_ptr + offs, mask=mask)
    y = tl.load(y_ptr + offs, mask=mask)
    tl.store(out_ptr + offs, x + y, mask=mask)
