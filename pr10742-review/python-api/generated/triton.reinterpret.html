triton.reinterpret

triton.reinterpret(tensor, dtype)

Reinterprets a tensor’s data as a different element type, without copying.

This is a bitcast-like view: the underlying bytes are unchanged and only their interpretation differs. Useful for element types that have no native framework dtype (for example the 8-bit floating-point formats).

Parameters:

  • tensor – the tensor whose data should be reinterpreted.

  • dtype – the Triton dtype to interpret the data as.

Returns:

a TensorWrapper viewing tensor as dtype (or the original tensor when the reinterpretation is a no-op).
