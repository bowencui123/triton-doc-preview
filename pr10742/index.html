Compilation

triton.compile(src, target=None, options=None)

Compiles a kernel to a device binary.

This is the lowering entry point used by triton.jit(); it can also be called directly for ahead-of-time compilation.

Parameters:

  • src – the kernel to compile, given either as an ASTSource (a @triton.jit function together with its signature) or as a path or string pointing at a textual IR module.

  • target – the GPUTarget to compile for; defaults to the target of the currently active device.

  • options – backend-specific compilation options (e.g. num_warps, num_stages) layered over the backend defaults.

Returns:

a compiled kernel that can be launched on the device.

class triton.AsyncCompileMode(executor, *, ignore_errors=False)

A context manager that compiles kernels asynchronously on an executor.

Inside the with block, kernel compilations triggered by launches are submitted to the given concurrent.futures.Executor and run in the background, each returning a FutureKernel. Any compiles still outstanding are finalized when the block exits.

class triton.FutureKernel

A handle to a kernel that is being compiled asynchronously.

Returned by AsyncCompileMode when a compilation is submitted. Call result() to block until the kernel is ready; attribute access on the handle also transparently waits for compilation to finish and forwards to the compiled kernel.

JIT

Unchanged: jit

class triton.JITFunction

A just-in-time compiled Triton kernel.

Instances are produced by applying the triton.jit() decorator to a Python function and are rarely constructed directly. A kernel is launched with the kernel[grid](*args, **kwargs) syntax: on the first launch for a given set of argument types the kernel is specialized and compiled, and the result is cached for subsequent launches.

Tensor Reinterpretation

triton.reinterpret(tensor, dtype)

Reinterprets a tensor’s data as a different element type, without copying.

This is a bitcast-like view: the underlying bytes are unchanged and only their interpretation differs. Useful for element types that have no native framework dtype (for example the 8-bit floating-point formats).

Parameters:

  • tensor – the tensor whose data should be reinterpreted.

  • dtype – the Triton dtype to interpret the data as.

Returns:

a TensorWrapper viewing tensor as dtype (or the original tensor when the reinterpretation is a no-op).

class triton.TensorWrapper

Presents a tensor under a different element type.

Returned by triton.reinterpret(). The wrapper forwards the storage, shape, strides, and device of the wrapped tensor while reporting dtype as the new type, so a kernel reads the same bytes under a different interpretation.

Memory

triton.set_allocator(allocator)

The allocator function is called during kernel launch for kernels that require additional global memory workspace.

Utilities

triton.cdiv(x, y)

Returns the ceiling division of x by y (that is, x / y rounded up to the nearest integer).

Commonly used on the host to size a launch grid, e.g. grid = (triton.cdiv(n_elements, BLOCK_SIZE), ).

Parameters:

  • x – the dividend.

  • y – the divisor.

triton.next_power_of_2(n)

Return the smallest power of 2 greater than or equal to n

Exceptions

exception triton.TritonError

Base class for all errors raised by Triton. (listed without a docstring in the PR — its generated page shows just Bases: Exception )

exception triton.CompilationError

Base class for all errors raised during compilation

exception triton.OutOfResources

Raised when a kernel needs more of a hardware resource than is available, for example more shared memory or registers than the device provides.

Reducing block sizes or num_stages often resolves it. The resource is named by name, with the requested and available amounts in required and limit.

exception triton.InterpreterError

Raised when an error occurs while executing a kernel in the interpreter (TRITON_INTERPRET=1).
